Security Architecture

Designed for enterprise CRM data boundaries.

How does PaceLock protect sensitive CRM records?

PaceLock follows least-privilege access patterns and reads only analytical parameters such as deal stage, timestamp metrics, amount, activity cadence, and governance state. It does not require raw customer email bodies or phone data for the core cockpit.

Are tenant datasets isolated?

Yes. The architecture uses tenant-scoped records, Postgres Row-Level Security, encrypted token storage, and strict session context for sync and forecasting queries.

Are internal notes used to train public models?

No. Unstructured notes are processed to populate structured JSON analysis fields. The workflow is designed to be stateless and does not use customer records for public model training.
